# SignalPilot — робочі правила агента

Постійні правила для будь-якої сесії роботи над цим проєктом. Читати першими.

## 1. Дозвіл на зміни
- Я не програміст — пояснюй простими словами, маленькими кроками.
- НЕ змінювати код без явного "погнали" / "працюємо" / прохання внести правки. Без цього — тільки обговорення.

## 2. Безпечні дії — робити ОДРАЗУ, без запитань
- Локальний `commit` документів, continuation-prompt, RIG/бектест-змін — робити САМОСТІЙНО, не питаючи дозволу. Це безпечно (офлайн).
- Кожну завершену порцію роботи фіксувати локальним комітом. **Commit = бекап.**
- НІКОЛИ не робити `push` без явного дозволу. Push/GitHub Actions/Telegram — окремий крок лише для ЖИВОЇ системи й лише за командою.

## 3. Жива система vs офлайн
- Жива система (`market-brief.yml`, Telegram, Move Alert): після правок — тести → commit → push → ручний Actions → Telegram → перевірка логів.
- RIG (`src/signalpilot/rig/`) — ОФЛАЙН вимірювання. Push/Actions/Telegram для нього НЕ потрібні; він не торкається того, що шлеться в Telegram.

## 4. Бекапи й відновлення (якщо трапилась халепа)
- **Останній чистий файл:** `git show HEAD:<шлях>` — друкує останню закомічену версію.
- **Повернути файл:** з git history. На монті D:\ `git checkout` може падати ("Operation not permitted" — заборонений unlink); тоді перезаписати файл через bash python `open('<шлях>','w',encoding='utf-8',newline='').write(<вміст з git>)`.
- **Надійна робоча копія для тестів:** локальний Linux fs `/tmp/sp` (монт глючить при виконанні).
- **Головне:** ми комітимо безпечну роботу ОДРАЗУ — тому останній коміт завжди і є бекап. Тому правило #2.

## 5. Монт D:\ глючить з інструментом редагування
- Edit-інструмент на монті може вставити null-байти у файл (він перестає імпортуватись).
- Тому код у `rig/` правити через bash, потім перевіряти `null bytes == 0` і ганяти тести. Можна працювати в `/tmp/sp`, далі синхронізувати на монт через python `open('w')`.
- Видалення файлів на монті треба окремо дозволити.
- Пісочниця має лише Python 3.10 (проєкт просить 3.11): запуск через `PYTHONPATH=src python3 ...`, не через `pip install -e .`.

## 6. RIG як ворота (правило продукту)
```
Rule: no strategy arm is considered working until it has a reproducible RIG
report against baseline. Reports must separate signal quality, fill mechanics,
and exit geometry where applicable.
```
Жодна торгова "рука" не робоча, поки немає відтворюваного RIG-звіту проти baseline.
